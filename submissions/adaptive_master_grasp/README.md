# Adaptive Master Grasp

**FF Master picks arbitrary objects without dropping or damaging them.**

Adaptive Master Grasp is a realistic manipulation submission centered on safe object pickup. The task is not to move a scripted block; it is to infer a grip strategy for an unknown item, close until measured contact is stable, lift, carry, and place while respecting object-specific damage limits.

The submission uses the repo's FF Master assets as the real robot reference and ships a task bench with a wrist-mounted adaptive gripper, declared MuJoCo sensors, randomized object properties, slip detection, and force limiting. The FF Master bottle demo now runs through an autonomous contact-servo rollout: phase changes are gated by sensed alignment, contact count, slip safety, and stable grasp steps rather than fixed timestamps.

## Robot Platform

- FF Master humanoid asset from `assets/Master` is compiled as part of the audit.
- Task bench: FF Master right-arm end-effector abstraction with a 6-DOF wrist carriage and four adaptive fingers.
- Sensors: wrist pose, joint position/velocity, per-finger touch, object pose, object velocity, grip force, slip margin, and damage margin.

## Task Goal

Pick up arbitrary everyday objects without dropping or damaging them:

- fragile egg,
- glass cup,
- foam cube,
- metal can,
- slippery bottle,
- deformable pouch,
- tool handle,
- irregular parcel.

Each object has different mass, friction, size, geometry class, and maximum safe grip force. The same controller must choose a grip mode, regulate force, recover slip, lift the object clear of the table, carry it, and place it down.

## Technical Approach

The controller runs a three-layer policy:

1. **Object-conditioned grasp selection** chooses pinch, enveloping, tripod, or cradle grip from size, mass, friction, and fragility estimates.
2. **Closed-loop force controller** increases normal force only until the measured slip margin is positive.
3. **Safety supervisor** caps force below the damage threshold, slows lift when slip margin is low, and aborts if a fragile object approaches damage.

The evaluation is domain randomized across object mass, friction, pose offset, and sensor noise. Every trial exports contact force, slip margin, damage margin, lift height, and final success.

## Core Features

- Arbitrary-object benchmark with eight object families.
- Real FF Master robot asset compile check.
- Adaptive grasp modes rather than one fixed grip.
- Slip recovery without exceeding damage limits.
- Fragile-object force limiting.
- 10-check audit for runnability, robot asset validity, sensors, lift, no-drop, no-damage, policy adaptation, robustness, and evidence generation.
- Demo video generated from submitted code.
- Autonomous bottle pickup evidence path using the same controller loop as the contact report.

## Current Limitations

This is a simulation submission. The FF Master full-body asset is used as the robot reference, while the grasp bench abstracts the right arm and end effector so the arbitrary-object policy is inspectable and reproducible. The bottle demo is closed-loop over MuJoCo contacts, but still uses a kinematic joint command layer because the provided FF Master asset does not expose a full actuated dexterous hand controller. No physical hardware numbers are claimed.

## Future Improvements

Attach the benchmark to the full FF Master arm IK chain, replace the gripper abstraction with hardware-specific fingertips, and fit the object property estimator from camera/tactile data.

## How To Run

```bash
cd submissions/adaptive_master_grasp
pip install -r requirements.txt
python3 run.py --quick
python3 run.py --eval
python3 run.py --demo
```

Generated artifacts are written to `artifacts/`.

## Demo Video

`artifacts/demo.mp4` is generated by `python3 run.py --demo`. The demo renders the FF Master MuJoCo model itself and a bottle object built from MJCF geoms. The `autonomous_contact_servo` controller advances through approach, close, lift, carry, and place from measured alignment and MuJoCo contact/slip metrics; the bottle follows the wrist only after stable grasp conditions are met. Live metrics show contact count, normal force, slip safety factor, crush margin, lift height, and trial state.
